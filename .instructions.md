# Umati Instructions

This is a living document for app-wide and game-specific design decisions.

## Global

- Never use emojis in product UI, documentation, or generated copy.
- Do not change existing text copy unless explicitly requested.
- Prefer mobile-first controls for player-facing game actions.
- Favor familiar iconography and obvious touch targets over hidden interactions.
- Full-screen surfaces must remain scrollable on mobile; do not rely on fixed desktop-height shells.

## Draw It

- The drawing toolbar should feel familiar to phone users: clear brush, eraser, fill, undo/clear actions, and a visible color palette.
- Brush size selection should use a compact popover, not a full-screen takeover.
- Active brush size should remain visible on the canvas while drawing.
- Include white and near-black in the palette, plus a broader range of saturated colors.
- Fill should behave as a bounded tap fill inside closed line art rather than painting the full canvas.
- Support multi-word prompts and show per-word lengths in the clue treatment.
- Do not add keyboard-shortcut affordances for Draw It because the primary interaction target is mobile.

## Polls

- Host poll creation and live results should use a full-screen surface, not a small modal.
- Poll screens should use a black base with frosted gradient treatments consistent with the lobby card language.
- Include quick-start poll suggestions for common host decisions, especially choosing the next game.
- Leaving the host poll screen via Back should close the active poll and return to the lobby.
